# 应用管理系统重构说明

## 主要变更

### 1. 数据存储方式改变
- **旧方式**: 从 `apps.json` 文件读取应用配置
- **新方式**: 用户自定义应用，数据存储在 localStorage 中（`user_apps` key）
- **优势**: 
  - 支持用户自定义添加应用
  - 数据持久化在本地
  - 为后期云备份预留接口

### 2. 移除分类功能
- 应用不再按分类组织
- 简化UI，所有应用平铺展示
- 收藏应用单独展示在顶部

### 3. 新增功能

#### 3.1 收藏应用
- 点击星号图标可收藏/取消收藏应用
- 收藏的应用会优先显示在"收藏应用"区域
- 收藏状态会持久化保存

#### 3.2 自定义创建应用
- 点击"添加应用"按钮打开创建表单
- 支持自定义：
  - 应用名称
  - 网址
  - 应用图标（支持上传 PNG、SVG、ICO 等格式）
  - 主题色

#### 3.3 编辑应用
- 每个应用卡片悬停时显示操作按钮
- 点击编辑按钮可修改应用信息
- 修改后自动保存

#### 3.4 删除应用
- 点击删除按钮，确认后删除应用
- 删除操作不可恢复

#### 3.5 拖拽排序
- 支持在同一区域内拖拽应用卡片重新排序
- 排序会自动保存
- 收藏应用和普通应用分别排序

### 4. 内置默认应用

系统内置以下默认应用（仅在首次使用时初始化）：
- Temu
- Telegram
- X (Twitter)
- 抖音
- 哔哩哔哩
- WhatsApp

### 5. MyAppsView 页面调整
- 移除分类显示
- 按应用 ID 分组显示浏览器配置
- 保留批量操作功能

## 文件变更

### 新增文件
- `src/utils/app-storage.ts` - 应用配置存储工具

### 修改文件
- `src/types/app-config.ts` - 更新 AppConfig 类型定义
- `src/components/AppGrid.svelte` - 完全重构，添加 CRUD 功能
- `src/components/MyAppsView.svelte` - 移除分类显示

## 数据结构

### AppConfig 接口
```typescript
interface AppConfig {
  id: string;              // 应用 ID（唯一标识）
  name: string;            // 应用名称
  url: string;             // 应用网址
  icon: string;            // 图标（base64 或 URL）
  color?: string;          // 主题色
  isFavorite: boolean;     // 是否收藏
  order: number;           // 排序顺序
  createdAt: number;       // 创建时间戳
  updatedAt: number;       // 更新时间戳
}
```

## 使用说明

### 添加新应用
1. 点击"添加应用"按钮
2. 填写应用信息
3. （可选）上传应用图标
4. 点击"创建"

### 收藏应用
- 点击应用卡片右上角的星号图标

### 编辑应用
1. 悬停在应用卡片上
2. 点击编辑按钮（笔形图标）
3. 修改信息后点击"保存"

### 删除应用
1. 悬停在应用卡片上
2. 点击删除按钮（×图标）
3. 确认删除

### 调整排序
- 直接拖拽应用卡片到目标位置
